# 실험 요약: M1 ~ M5

> 이 문서는 LatentVLA 프로젝트를 전혀 모르는 사람도 읽을 수 있게 작성된 요약입니다.

---

## 1. 이 연구가 뭔가요?

로봇이 시연 데이터로부터 **조작 작업을 학습하는 방법**을 연구합니다.

목표는 단순합니다: 사람이 로봇 팔로 "컵을 집어서 선반에 올리는" 장면을 몇 번 보여주면, 로봇이 스스로 같은 동작을 따라할 수 있게 만드는 것입니다.

---

## 2. 기존 연구의 문제점

최근 잘 나가는 로봇 학습 모델들(π0, OpenVLA 등)은 다음과 같이 작동합니다:

```
카메라 이미지 + 언어 지시 → [거대한 VLM 모델] → 관절 각도
```

이 방식은 **"뭘 해야 하나"** 와 **"어떻게 움직이나"** 가 한 모델 안에 뒤섞여 있습니다.

| 연구 | 문제점 |
|------|--------|
| π0 | VLM 토큰을 그냥 행동 생성에 연결. "계획"이라는 개념 자체가 없음 |
| π0.5 | 텍스트로 subtask를 명시하지만, 중간 표현이 학습되지 않음 |
| Fast-in-Slow | VLM의 hidden state를 그대로 사용. z를 설계하지 않았고, 품질도 측정 안 함 |
| RationalVLA | `<ACT>` 토큰 하나로 z를 대표 → 너무 단순, 표현력 한계 |

**아무도 묻지 않은 질문이 있습니다:**

> "z(계획 벡터)의 품질이 좋아지면 행동이 실제로 좋아지는가? 어떤 z가 좋은 z인가?"

---

## 3. 우리의 아이디어: LDM 비유

이미지 생성 AI의 패러다임 변화와 같은 방식입니다.

**이미지 생성의 역사:**
```
옛날: 픽셀 공간에서 직접 Diffusion → 느리고 비효율
LDM(Stable Diffusion): VAE로 이미지를 압축 → 작은 latent 공간에서 Diffusion
              → 훨씬 빠르고 품질 좋음
```

**로봇 행동 생성에 같은 원리 적용:**
```
기존 VLA: 행동 공간에서 직접 Flow/Diffusion
           → "뭘 해야 하나"와 "어떻게 움직이나"가 뒤섞임

우리 제안: VLM이 "계획 z"를 만들고 → z 공간에서 행동 Flow
           → "뭘 해야 하나"(시스템 2)와 "어떻게 움직이나"(시스템 1)가 분리됨
```

## 4. 시스템 1 vs 시스템 2

심리학자 Daniel Kahneman의 사고 이론에서 가져온 개념입니다.

| | 시스템 2 (느린 사고) | 시스템 1 (빠른 사고) |
|--|---------------------|---------------------|
| **특징** | 논리적, 계획적, 느림 | 직관적, 반사적, 빠름 |
| **사람 예시** | 체스 수 계산, 요리 순서 계획 | 자전거 타기, 공 받기 |
| **로봇 역할** | "이 상황에서 뭘 먼저 해야 하나?" | "팔을 정확히 어디로 움직이나?" |
| **우리 모델** | VLM (PaliGemma-3B) | Flow 기반 행동 디코더 |

```
[카메라 이미지 + 언어 지시 + 관절 상태]
            ↓
    ┌──────────────────┐
    │   시스템 2 (VLM)  │  ← 느리지만 "뭘 할지" 이해
    │  PaliGemma-3B    │
    └────────┬─────────┘
             ↓
         z (계획 벡터)
         "컵이 왼쪽, 선반은 오른쪽 위,
          먼저 컵을 향해 접근해야 함"
             ↓
    ┌──────────────────┐
    │  시스템 1 (Flow)  │  ← 빠르게 구체적 동작 생성
    │  행동 디코더      │
    └────────┬─────────┘
             ↓
    관절 각도 × 16 스텝
```

**M1~M4 실험은 시스템 1(Flow)은 고정하고, z를 어떻게 만드느냐만 바꿔서 실험합니다.**
(VLM 기반 시스템 2는 M5 LatentVLA에서 등장)

## 5. 핵심 개념 쉽게 이해하기

### "Prior"와 "Posterior"란?

수학 용어가 어렵게 느껴지지만, 아이디어는 단순합니다.

**상황을 생각해봅시다:**
- 로봇이 지금 카메라로 현재 상황을 보고 있습니다
- 로봇은 "앞으로 어떻게 될지" (미래)를 모릅니다

```
Prior (사전 지식만 있을 때):
  지금 보이는 것 + 경험 → z 추측
  "컵이 왼쪽에 있는 것 같은데, 아마 컵을 집으러 가야겠지?"
  → 추론 시(실제 로봇 작동) 이 방법만 쓸 수 있음

Posterior (미래 정보도 알 때):
  지금 보이는 것 + 미래에 어떻게 됐는지 → z 계산
  "컵을 집었고, 그 다음 선반에 올렸구나. z는 이거였네."
  → 학습 시에만 쓸 수 있음 (미래 = 정답 레이블)
```

**비유:** 시험 전 공부 vs 시험 후 정답지 보기
- Prior = 시험 전에 공부한 것만으로 문제 풀기 (실제 상황)
- Posterior = 정답지를 보면서 문제 푸는 것 (이상적 상한선)

**MSE (Mean Squared Error):**
- 행동 예측이 얼마나 틀렸는지를 수치로 표현
- 낮을수록 좋음

따라서:
- **MSE (prior)**: 미래 모르고 예측한 오차 → **실제 성능**
- **MSE (posterior)**: 미래 알고 예측한 오차 → **이론적 최고 성능**

---

### "z_shuffle_gap"이란?

이 지표는 **"z가 얼마나 중요한 정보를 담고 있냐"** 를 측정합니다.

**실험 방법:**
1. 정상적으로 z를 사용해서 행동 예측 → 오차 측정
2. z를 다른 샘플(다른 작업)의 것과 무작위로 바꿔서 예측 → 오차 측정
3. **z_shuffle_gap = (섞은 후 오차) - (정상 오차)**

```
예시:
  "컵 집기" 작업의 z를 "서랍 열기" 작업의 z와 바꾸면?

  z가 의미 있다면: 행동이 엉망이 됨 → 오차 크게 증가 → gap 큼
  z가 의미 없다면: 어차피 z는 노이즈였으니 별 차이 없음 → gap 작음
```

**결론:** z_shuffle_gap이 클수록 z가 작업에 핵심적인 정보를 담고 있음

---

### "prior_posterior_gap"이란?

이 지표는 **"z를 더 잘 만들면 얼마나 좋아질 수 있냐"** 를 나타냅니다.

```
prior_posterior_gap = MSE(prior) - MSE(posterior)
```

- gap이 크다 → posterior(정답 z)와 prior(추측 z) 성능 차이가 큼
  → "z를 더 잘 만들 수 있으면 성능이 크게 오를 여지가 있다"
  → z가 성능에 중요한 역할을 하고 있다는 뜻

- gap이 작다 → prior ≈ posterior
  → "z를 잘 만들어도 별로 좋아지지 않음" = z가 사실 별로 안 중요함

---

### 전체 지표 요약

| 지표 | 직관적 의미 | 좋은 방향 |
|------|------------|----------|
| **MSE (prior)** | 실제 추론 시 행동 예측 오차 | 낮을수록 ↓ |
| **MSE (posterior)** | 미래 정보 알 때 최고 성능 (상한선) | 낮을수록 ↓ |
| **z_shuffle_gap** | z가 얼마나 의미 있는 정보를 담고 있나 | 높을수록 ↑ |
| **prior_posterior_gap** | z를 더 잘 만들면 얼마나 좋아질 수 있나 | 높을수록 ↑ (개선 여지) |
| **best_of_5** | z를 5번 뽑아서 가장 좋은 것의 오차 | 낮을수록 ↓ |

## 6. 우리의 연구 질문과 가설

### 연구 질문 (Research Questions)

**RQ1 (핵심):** z의 품질이 행동 생성 품질을 실제로 결정하는가?
> z_shuffle_gap이 높은 모델이 실제 로봇 성공률도 높은가?

**RQ2:** VLM이 만든 z는 단순 MLP가 만든 z보다 품질이 높은가?
> PaliGemma-3B의 z가 MLP의 z보다 z_shuffle_gap과 성공률 모두에서 이기는가?

**RQ3:** z를 VLM에서 어떻게 뽑느냐도 중요한가?
> last token / mean pool / plan token 중 어떤 방식이 가장 좋은 z를 만드는가?

### 가설 (Hypotheses)

**H1:** z_shuffle_gap이 높을수록 실제 로봇 성공률이 높다
→ M1~M4 실험으로 검증 (이번 실험의 목적)

**H2:** VLM-z > MLP-z (품질 기준)
→ VLM은 언어와 시각 이해 능력으로 더 의미 있는 z를 만들 수 있다

**H3:** plan_token z > mean_pool z > last_token z
→ 계획을 요약하도록 설계된 특수 토큰이 가장 압축된 의미를 담는다

## 7. 이번 실험의 위치 (큰 그림)

```
이번 실험 (M1~M4)
  ↓
목적: "z 품질 → 행동 품질" 상관관계 증명 (Aha Moment 1)
  ↓
결과가 좋으면: "그럼 VLM으로 더 좋은 z를 만들자" 동기 확립
  ↓
다음 실험 (M5 LatentVLA)
  ↓
목적: VLM-z vs MLP-z 비교 증명 (Aha Moment 2)
```

**M1~M4는 논문 전체를 정당화하는 기반 실험입니다.**
이 실험이 없으면 "왜 VLM으로 z를 개선해야 하냐"는 질문에 답할 수 없습니다.

## 8. 모델 설명

| ID | 모델 | z 생성 방법 | 한 줄 설명 |
|----|------|------------|------------|
| **M1** | FlatFlow | z 없음 | 컨텍스트 → 행동 직접 생성. 가장 단순한 베이스라인 |
| **M2** | DetLatent | MLP(미래 이미지 피처) | 학습 시 미래 정보로 z 계산. 결정론적 (z가 하나로 고정) |
| **M3** | StochVAE | VAE posterior 샘플링 | z를 확률 분포로 표현. KL 정규화로 prior 학습 |
| **M4** | StochFlowPrior | 플로우 기반 prior 샘플링 | VAE 대신 정규화 플로우로 더 복잡한 prior 표현 |
| **M5** | VLM SFP Plan | VLM plan token → StochFlowPrior | PaliGemma-3B의 plan token으로 z 생성. 시스템 2(VLM) + 시스템 1(Flow) 통합 |

### M1 — "z 없이 해보자"
```
현재 상태 → [컨텍스트 인코더] → C_t → [Flow 디코더] → 행동
```
가장 단순. z 개념 자체가 없음. 다양한 미래를 표현 불가.

### M2 — "정답 미래로 z 만들기"
```
학습 시: 미래 이미지 → [SigLIP] → [MLP] → z (고정값)
추론 시: z를 컨텍스트에서 예측해야 함 (미래 이미지 없음)
```
z 품질이 이론적으로 가장 높지만, 학습과 추론 간 gap이 존재.

### M3 — "VAE로 z 분포 학습"
```
학습: C_t + 미래 피처 → posterior 분포 → z 샘플링
추론: C_t → prior 분포 → z 샘플링
KL loss로 prior가 posterior를 흉내내도록 학습
```
문제: KL 정규화가 너무 강하면 z가 의미 없는 가우시안 노이즈가 됨 (KL collapse).

### M4 — "플로우로 더 표현력 있는 prior"
```
VAE의 가우시안 prior → 정규화 플로우로 교체
더 복잡한 분포를 표현 가능 → KL collapse 일부 완화
```

## 9. 실험 결과 (100 epoch, LIBERO-Object 데이터셋)

| 모델 | MSE (prior) ↓ | MSE (posterior) ↓ | z_shuffle_gap ↑ | prior_posterior_gap ↑ | best_of_5 ↓ |
|------|:-------------:|:-----------------:|:---------------:|:---------------------:|:-----------:|
| M1 FlatFlow       | 0.5530 | —      | —          | —      | —      |
| M2 DetLatent      | **0.4776** | **0.0017** | **0.7837** | **0.4759** | —      |
| M3 StochVAE       | 0.6321 | 0.6205 | -0.0039    | 0.0116 | 0.3180 |
| M4 StochFlowPrior | 0.6540 | 0.5395 | 0.0428     | 0.1144 | 0.3638 |
| M5 VLM SFP Plan   | 0.6084 | 0.5318 | 0.0163     | 0.0766 | **0.3645** |

---

## 10. 결과 해석

### M1 (FlatFlow) — 베이스라인
z 없이도 MSE 0.553. 그러나 다양한 미래를 표현할 수 없어 한계가 명확합니다.

### M2 (DetLatent) — "z가 좋으면 성능이 얼마나 좋아지는지" 증명
- `mse_posterior = 0.0017` → **정답 z를 알면 거의 완벽한 행동 예측 가능**
- `z_shuffle_gap = 0.784` → z를 섞으면 성능이 크게 하락 = **z가 작업의 핵심 정보를 모두 담고 있음**
- **이것이 이 연구의 핵심 동기**: "좋은 z만 있으면 문제가 풀린다"
- **한계**: 실제 추론 시 미래 이미지가 없으므로 z를 컨텍스트에서만 추측해야 함

### M3 (StochVAE) — VAE의 실패
- `z_shuffle_gap = -0.004` ≈ 0 → **z를 섞어도 성능이 거의 안 변함 = z가 무의미**
- `prior_posterior_gap = 0.012` ≈ 0 → prior ≈ posterior = 모델이 z를 무시하고 행동 생성
- KL 정규화가 z를 단순 가우시안 노이즈로 만들어버림 (KL collapse)

### M4 (StochFlowPrior) — VAE보다 낫지만 아직 부족
- `z_shuffle_gap = 0.043` → M3(-0.004)보다 훨씬 낫지만 M2(0.784)에는 크게 못 미침
- `prior_posterior_gap = 0.114` → z가 어느 정도 활용되고 있음
- `best_of_5 = 0.364` → 여러 번 샘플링해서 최선 선택 시 이득 있음

### M5 (VLM SFP Plan) — VLM z의 첫 시도
- `mse_prior = 0.608` → M4(0.654)보다 개선. VLM z가 M4보다 나은 prior 추론
- `mse_posterior = 0.532` → M4(0.540)와 유사한 수준
- `z_shuffle_gap = 0.016` → M4(0.043)보다 낮음. VLM z가 작업 정보를 충분히 담지 못함
- `prior_posterior_gap = 0.077` → M4(0.114)보다 작음. prior-posterior 정렬이 비교적 잘 됨
- `best_of_5 = 0.365` → M3(0.318)보다 높고 M4(0.364)와 유사

**해석:** VLM plan token을 z로 사용하면 MSE prior 측면에서 M4보다 개선되지만, z_shuffle_gap은 오히려 낮아짐. VLM z가 flow prior와 잘 통합됐지만, plan token이 작업 구분 정보를 충분히 분리하지 못할 가능성. 더 나은 VLM z 추출 방식 탐색이 필요.

### 결론 (Aha Moment 1 확인 + M5 추가)
```
z_shuffle_gap:  M3(-0.004)  <  M5(0.016)  <  M4(0.043)  <<  M2(0.784)
MSE (prior):    M2(0.477)   <  M1(0.553)  <  M5(0.608)  <  M3(0.632) < M4(0.654)
```

> **z 품질이 좋을수록 행동 품질이 좋아진다** — M2가 이를 명확히 증명.
> M5(VLM)는 MSE prior 개선에 성공했으나, z_shuffle_gap은 M4보다 낮아 VLM z의 작업 정보 표현력 개선이 과제.

## 11. 앞으로의 방향

M5 실험 결과를 바탕으로 두 가지 과제가 명확해졌습니다.

**과제 1: VLM z의 z_shuffle_gap이 M4보다 낮은 이유 규명**
- plan token이 작업 구분 정보를 충분히 포착하지 못할 가능성
- VLM fine-tuning 없이 frozen 상태로 z를 추출한 것이 원인일 수 있음

**과제 2: RQ3 미수행 — z 추출 방식 비교 실험 필요**
- 현재 M5는 plan token 방식 하나만 실험됨
- last_token / mean_pool / plan_token 세 방식의 직접 비교 필요

| M5-last | VLM + SFP | last token (기본 VLM 출력) | 베이스라인 |
| M5-mean | VLM + SFP | mean pool (전체 토큰 평균) | 정보 분산 효과 |
| M5-plan | VLM + SFP | plan token (현재 M5) | 현 결과 재확인 |

**목표 지표:** z_shuffle_gap이 M4(0.043)를 넘는 방식 확인

**커맨드 참고:**
```bash
# M5-last
torchrun --nproc_per_node=<N> train.py --config configs/vlm_sfp_last.yaml

# M5-mean
torchrun --nproc_per_node=<N> train.py --config configs/vlm_sfp_mean.yaml
```

---

### Step 2. VLM fine-tuning 실험 — RQ2 본격 검증

현재 M5는 PaliGemma frozen 상태. VLM을 task-specific하게 fine-tune하면 z_shuffle_gap이 개선될 것으로 예상.

**방향:**
- z 추출 레이어에 contrastive loss 추가 (작업이 다른 샘플의 z는 멀어지도록)
- 또는 VLM last N layer unfreeze 후 end-to-end 학습

**성공 기준:** z_shuffle_gap > M4(0.043), 이상적으로 > 0.2 수준

---

### Step 3. 실제 로봇 평가 — H1 검증 (크리티컬)

현재 모든 평가는 MSE 기반. 실제 로봇 성공률 없이는 논문 주장(H1) 검증 불가.

| 모델 | z_shuffle_gap | 예상 성공률 | 비고 |
|------|:------------:|:-----------:|------|
| M1 FlatFlow       | —      | 베이스라인 | |
| M2 DetLatent      | 0.784  | 최고 예상  | oracle z |
| M4 StochFlowPrior | 0.043  | M2보다 낮음 | |
| M5 VLM SFP Plan   | 0.016  | M4와 유사? | 확인 필요 |

**H1 검증 조건:** z_shuffle_gap 순서와 실제 성공률 순서가 일치하면 → "z 품질이 행동 품질을 결정한다" 입증

### Step 4. 장기 학습 (200ep) — 현재 결과가 수렴 전일 가능성

M4, M5 모두 100ep에서 z_shuffle_gap이 낮음. 학습이 더 필요한지 곡선으로 확인 후 결정.
