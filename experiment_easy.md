## 1. 이 연구가 풀려는 문제가 뭔가요?

### 로봇 팔이 물건을 잡으려면 무슨 일이 일어나야 할까요?

상상해보세요. 여러분 앞에 테이블이 있고, 그 위에 빨간 컵이 있어요. 누군가 "저 빨간 컵 집어줘" 라고 말합니다.

사람은 이걸 순식간에 해요:
1. 눈으로 빨간 컵의 위치와 모양을 파악하고
2. "빨간 컵 집어"라는 말의 의미를 이해하고
3. 팔을 어떻게 움직여야 할지 계획을 세우고
4. 실제로 팔과 손가락을 순서대로 움직여서 잡는다

로봇도 이 과정을 똑같이 해야 합니다. 특히 **3번 — 팔을 어떻게 움직일지 결정하는 것** 이 이 연구의 핵심입니다.

### 왜 어려운가요?

사실 "컵을 잡는 방법"은 단 하나가 아닙니다.

- 위에서 잡을 수도 있고
- 옆에서 잡을 수도 있고
- 천천히 잡을 수도 있고
- 빠르게 잡을 수도 있어요

즉, 하나의 목표(컵 잡기)에 대해 **수백 가지의 유효한 행동 시퀀스**가 존재합니다. 이것을 **다봉 분포(multimodal distribution)** 라고 불러요.

단순한 모델은 "평균적인 방법 하나"만 배웁니다. 그러면 이도 저도 아닌 어정쩡한 움직임이 나올 수 있어요. 이 연구는 **여러 유효한 방법을 동시에 배우고, 상황에 맞게 그 중 하나를 선택해 실행할 수 있는 모델**을 만드는 것이 목표입니다.

### 이 연구의 핵심 질문

> "다양한 행동을 만들어내려면, 모델 내부의 '잠재 변수(latent variable, z)'를 어떻게 설계해야 할까?"

4가지 서로 다른 설계 방법을 실험하고, 어떤 게 더 좋은지 비교합니다.

## 2. 핵심 도구: Flow Matching이 뭔가요?

### 생성 모델이란?

생성 모델은 "없는 것을 만들어내는" 모델입니다. 그림 AI처럼 텍스트를 주면 이미지를 만들어내듯이, 이 연구에서는 **현재 상황(이미지 + 언어 지시 + 로봇 자세)을 주면 앞으로 8 스텝 동안의 로봇 행동 시퀀스를 만들어냅니다.**

### 아이디어: 노이즈에서 데이터로

Flow Matching의 핵심 아이디어는 매우 직관적입니다.

```
[출발] 순수한 랜덤 노이즈   →→→→→→   [도착] 실제 로봇 행동 시퀀스
         (가우시안 분포)                    (학습 데이터)
```

마치 "흐릿하고 랜덤한 점들의 구름"에서 시작해서, 조금씩 방향을 잡아가며 결국 "의미 있는 행동 시퀀스"에 도달하는 것입니다.

### 비유: 물이 흐르는 것처럼

모델은 **속도 함수 v(x, t)** 를 배웁니다. 이것은 어느 시점 t에, 어느 위치 x에 있는 '입자(행동 시퀀스)'를 어느 방향으로 얼마나 빠르게 밀어야 최종 목적지(진짜 행동)에 도달하는지를 알려줍니다.

```
t=0: 랜덤 노이즈 ε
  ↓  (속도 v 적용)
t=0.1: 조금 움직임
  ↓
t=0.2: 조금 더 움직임
  ↓  ...10번 반복...
t=1.0: 완성된 행동 시퀀스 a
```

### 학습할 때는?

학습 시에는 정답 행동을 알고 있으니까, "이 중간 지점 t에서, 이 방향(정답 - 노이즈)으로 밀면 정답에 도달한다"는 것을 학습합니다.

수식으로 쓰면:
```
xt = (1-t) * 노이즈 + t * 정답_행동    (두 지점을 직선으로 보간)
목표 속도 = 정답_행동 - 노이즈           (출발점에서 도착점 방향)
손실 = ||모델이_예측한_속도 - 목표_속도||²
```

이게 바로 **OT-CFM (Optimal Transport Conditional Flow Matching)** 입니다.

### 왜 Diffusion 모델 대신 Flow Matching?

- Diffusion은 수백 ~ 수천 스텝이 필요하지만, Flow Matching은 **10~20 스텝**이면 충분합니다
- 경로가 직선에 가까워서 훨씬 효율적입니다
- 로봇처럼 실시간 응답이 중요한 분야에서 유리합니다

## 3. 4가지 모델 실험 — 각각 뭐가 다른가요?

모든 모델의 공통점:
- 같은 데이터를 사용합니다
- 같은 인코더(SigLIP)로 이미지와 언어를 이해합니다
- Flow Matching으로 행동을 생성합니다

차이점은 **"행동을 생성하기 전에 중간 잠재 변수 z를 어떻게 다루냐"** 입니다.

```
M1 FlatFlow:       상황 → [Flow] → 행동          (z 없음)
M2 DetLatent:      상황 → z(결정론적) → [Flow] → 행동
M3 StochVAE:       상황 → z(Gaussian 샘플) → [Flow] → 행동
M4 StochFlowPrior: 상황 → [Flow1] → z → [Flow2] → 행동
```

쉽게 말하면:
- **M1**: 바로 행동을 만든다
- **M2**: 계획(z)을 하나 세우고, 그 계획에 따라 행동을 만든다
- **M3**: 계획(z)을 확률적으로 랜덤하게 샘플링하고, 그 계획에 따라 행동을 만든다
- **M4**: 계획(z) 자체도 Flow로 생성하고, 그 계획으로 행동도 Flow로 만든다

## 4. 공통 재료: 컨텍스트 인코더

모든 모델이 공유하는 "상황 이해기"입니다.

### 입력

1. **이미지**: 카메라로 찍은 현재 장면 (224×224 픽셀)
2. **언어 지시**: "빨간 컵을 집어" 같은 텍스트
3. **고유감각(proprioception)**: 로봇 팔의 현재 관절 각도, 그리퍼 상태 등

### 처리 과정

```
이미지 ──→ SigLIP 이미지 인코더 (frozen, 학습 안 함)
언어  ──→ SigLIP 텍스트 인코더 (frozen, 학습 안 함)
                 ↓ 두 임베딩 합침(fusion)
고유감각 ──→ MLP 레이어
                 ↓
      컨텍스트 벡터 C_t (256차원)
```

**왜 SigLIP을 frozen으로 사용하나요?**

SigLIP은 구글에서 수억 장의 이미지-텍스트 쌍으로 학습한 모델입니다. 이미 "빨간 컵"이 뭔지, "집는다"가 뭔지 잘 알아요. 이걸 처음부터 다시 가르치는 것보다, 그냥 "이미 잘 알고 있는 선생님"처럼 이용하는 게 효율적입니다.

**컨텍스트 벡터 C_t가 담는 것**

256차원의 숫자 벡터 하나에, "지금 눈 앞에 빨간 컵이 있고, 집으라는 지시가 있고, 로봇 팔은 이런 자세"라는 모든 정보가 압축되어 있습니다.

## 5. 모델 1: FlatFlow (베이스라인)

### 한 줄 요약

> "상황만 보고 바로 행동 시퀀스를 만들어낸다. 중간 계획 따위 없다."

### 어떻게 작동하나요?

```
컨텍스트 C_t (256차원)
        ↓
   VelocityMLP (속도 예측 네트워크)
        ↓
  [Flow Matching 실행 - 10 스텝]
        ↓
  행동 시퀀스 (8 스텝 × 7차원)
```

VelocityMLP는 아래 세 정보를 합쳐서 "이 방향으로 가야 해"를 예측합니다:
- 현재 행동 위치 x_t
- 시간 t (sinusoidal 임베딩으로 변환)
- 컨텍스트 C_t

### 장점과 한계

**장점**: 구조가 단순하고 빠르다.

**한계**: 잠재 변수 z가 없으니, "다양한 해결 방법" 중 하나를 선택할 수 있는 메커니즘이 없습니다.

컵을 잡는 방법이 3가지가 있는데, 모델은 그 3가지를 평균 낸 "중간 어딘가" 행동을 배울 수 있어요. 이것이 실제로 실패하는 이유입니다.

### 학습 손실

```
전체 손실 = Flow Matching 손실
          = ||예측_속도 - (정답_행동 - 노이즈)||²
```

단 하나의 손실만 있어서 학습이 가장 단순합니다.

## 6. 모델 2: DetLatent (결정론적 잠재 변수)

### 한 줄 요약

> "상황을 보고 '어떤 방식으로 잡을지' 계획(z)을 먼저 하나 세운다. 그 계획에 맞는 행동을 만든다."

### 비유: 요리사의 레시피

요리사가 재료(상황)를 보고, 먼저 머릿속으로 "오늘은 볶음 스타일로 하겠다(z)"고 결정합니다. 그리고 그 결정에 맞춰 실제 요리 동작들(행동 시퀀스)을 수행합니다.

### 어떻게 작동하나요?

**학습 시 (선생님 모드):**
```
컨텍스트 C_t + 정답 행동 + 미래 이미지
        ↓ Posterior Encoder (MLP 3층)
        z* (128차원, 정답 행동을 압축한 잠재 변수)
        ↓
(C_t, z*) → VelocityMLP → 행동 생성
```

**추론 시 (혼자 하는 모드):**
```
컨텍스트 C_t
        ↓ Prior Head (MLP 2층)
        z_hat (128차원, 상황만 보고 추측한 계획)
        ↓
(C_t, z_hat) → VelocityMLP → 행동 생성
```

### 핵심 아이디어: 선생님-학생 구조

- **Posterior Encoder** = 선생님: 정답을 보고 z*를 만든다 (학습 때만 사용)
- **Prior Head** = 학생: 정답 없이 상황만 보고 z_hat을 예측한다 (실제 사용)
- 학습 목표 중 하나: Prior Head가 Posterior Encoder의 결과를 흉내 낼 것

```
Prior 손실 = MSE(z_hat, z*)  ← z_hat이 z*와 최대한 같아지도록
```

### z는 뭘 배울까요?

128차원의 벡터인데, 학습을 통해 자연스럽게 "이 상황에서 어떤 방식의 행동을 할지"를 압축하게 됩니다. 예를 들어:
- z[0] 이 크면 → 위에서 잡는 방식
- z[0] 이 작으면 → 옆에서 잡는 방식

이런 식의 의미 있는 구조가 자발적으로 형성됩니다 (해석하기 어렵지만).

### 한계

Prior Head가 **딱 하나의 z_hat**만 출력합니다. 잡는 방법이 3가지여도, 항상 "가장 그럴듯한 하나"만 선택합니다. 다양성이 없어요.

## 7. 모델 3: StochVAE (확률론적 VAE)

### 한 줄 요약

> "계획(z)을 확률 분포로 배운다. 매번 샘플링할 때마다 다른 z가 나와서 다양한 행동이 나온다."

### 비유: 요리사가 주사위를 던진다

같은 재료를 보고도, 주사위를 던져서 "1~3이면 볶음, 4~6이면 찜"으로 결정하는 것처럼, z를 가우시안 분포에서 **랜덤 샘플링**합니다.

### VAE란?

VAE(Variational AutoEncoder)는 데이터를 **확률 분포의 파라미터(평균 μ, 분산 σ²)** 로 인코딩하는 모델입니다. 잠재 변수 z를 `N(μ, σ²)` 에서 샘플링해서 사용합니다.

```
z ~ N(μ, σ²)  ← 매번 다른 값이 나옴
```

### 어떻게 작동하나요?

**학습 시:**
```
컨텍스트 C_t + 정답 행동 + 미래 이미지
        ↓ Posterior Encoder (GaussianEncoder)
        μ_q, σ²_q  ← 사후 분포(posterior) 파라미터
        ↓ 재파라미터화(reparameterization)
        z = μ_q + σ_q * ε   (ε ~ N(0,I))
        ↓
(C_t, z) → VelocityMLP → 행동 생성
```

동시에 Prior도 학습:
```
컨텍스트 C_t만
        ↓ Prior Encoder (GaussianEncoder)
        μ_p, σ²_p  ← 사전 분포(prior) 파라미터
```

**추론 시:**
```
컨텍스트 C_t → Prior → μ_p, σ²_p
        ↓ 샘플링
        z = μ_p + σ_p * ε   (매번 랜덤하게 달라짐!)
        ↓
(C_t, z) → VelocityMLP → 다양한 행동 생성
```

### KL 손실이란?

VAE의 핵심 학습 신호입니다.

```
KL 손실 = KL(q(z|C,a) || p(z|C))
```

이게 의미하는 것: "학습 때 사용하는 Posterior 분포와, 실제로 쓸 Prior 분포가 최대한 비슷해야 한다"는 제약입니다.

왜 필요하냐면: Prior는 정답 행동을 못 보니까, Posterior가 너무 다른 분포를 배우면 추론 때 Prior로 샘플링한 z가 학습 때 z와 달라서 엉망이 됩니다.

### Posterior Collapse 문제

VAE의 유명한 단점입니다. 학습하다 보면 KL 손실을 줄이려고 **모델이 z를 무시하고 μ_q ≈ μ_p, σ_q ≈ σ_p 로 만들어버립니다.** 그러면 z는 아무 의미 없는 노이즈가 되고, 모델은 z를 무시하고 컨텍스트만으로 행동을 예측하게 됩니다. 다양성이 사라지는 거예요.

β-VAE처럼 KL 가중치(β)를 조절해도 쉽지 않습니다.

## 8. 모델 4: StochFlowPrior (제안 모델)

### 한 줄 요약

> "계획(z) 자체도 Flow로 만들고, 행동도 Flow로 만든다. KL 없이도 다양성을 유지한다."

### 핵심 아이디어

M3의 Posterior Collapse 문제를 해결하면서 다양성을 유지하려면 어떻게 해야 할까요?

**해결책: Prior를 Gaussian이 아니라 Flow Matching으로 만들자!**

```
기존 Prior (M3): z ~ N(μ_p, σ²_p)  ← Gaussian, 간단하지만 표현력 한계
새로운 Prior (M4): ε ~ N(0,I) →→→[Flow1]→→→ z  ← Flow, 훨씬 복잡한 분포 표현 가능
```

### 비유: 이중 요리 과정

요리 레시피를 만드는 것(z 생성)도, 실제 요리를 하는 것(행동 생성)도, 둘 다 "처음에 막연한 아이디어에서 시작해서 구체화하는 과정"입니다.

```
막연한 영감(노이즈 ε)
        ↓ [Flow 1: 요리 스타일 결정 과정]
  구체적 레시피 계획(z)
        ↓ [Flow 2: 실제 요리 행동 생성]
  실제 요리 동작 시퀀스(a)
```

### 어떻게 작동하나요?

**학습 시:**
```
스텝 1: Posterior z* 구하기
    컨텍스트 C_t + 정답 행동 + 미래 이미지
            ↓ GaussianEncoder
            z* (정답 행동이 담긴 잠재 변수)

스텝 2: 행동 Flow 학습 (z*를 조건으로)
    행동 Flow: ||v_action(a_t, t, [C_t, z*]) - 목표 속도||²

스텝 3: 잠재 Flow 학습 (z*를 목표로)
    잠재 Flow: ||v_prior(ε_t, t, C_t) - (z* - ε)||²
    즉, "노이즈에서 z*로 가는 경로"를 배움
```

**추론 시:**
```
컨텍스트 C_t
        ↓ [Flow 1: prior_flow 실행 - 10스텝]
        z (다양한 z가 샘플링됨!)
        ↓
(C_t, z) → [Flow 2: action_flow 실행 - 10스텝]
        ↓
  다양한 행동 시퀀스
```

### KL이 없어도 되는 이유

VAE에서 KL 손실이 필요한 이유는: "Posterior와 Prior가 같은 공간에서 작동하게" 강제하기 위해서였습니다.

Flow Prior에서는 다릅니다:
- Prior Flow는 z*를 직접 학습 목표로 삼아 "어떻게 하면 Posterior의 z* 분포를 재현할 수 있는지" 경로를 배웁니다
- Gaussian 가정 없이 임의의 복잡한 분포를 표현할 수 있습니다
- Posterior Collapse가 일어나기 어렵습니다

### 시맨틱 보조 손실 (Semantic Auxiliary Loss)

M4 (그리고 M2, M3)에는 추가적인 학습 목표가 있습니다:

```
"z를 보고 미래 이미지의 특징을 예측할 수 있어야 한다"
```

구체적으로:
```
z → semantic_head (Linear 레이어) → 예측된 미래 임베딩
손실 = 1 - cosine_similarity(예측된 미래 임베딩, 실제 미래 이미지 임베딩)
```

왜 이걸 하냐면: z가 "어디로 이동하는지, 결과적으로 무엇을 달성하는지"와 같은 **의미 있는 정보**를 담도록 유도하기 위해서입니다. z가 단순히 "행동 패턴"만 담는 게 아니라 "목표 상태"도 함께 담도록 해서, 더 계획적인 행동 생성이 가능해집니다.

### 요약 비교표

| 특징 | M1 FlatFlow | M2 DetLatent | M3 StochVAE | M4 StochFlowPrior |
|------|------------|-------------|------------|------------------|
| 잠재 변수 z | 없음 | 있음 (결정론적) | 있음 (Gaussian) | 있음 (Flow) |
| 다양성 | 없음 | 없음 | 있음 | 있음 |
| Prior 표현력 | — | 낮음 (MLP→점) | 중간 (Gaussian) | 높음 (Flow→임의 분포) |
| KL 손실 | 없음 | 없음 | 있음 | 없음 |
| Posterior Collapse 위험 | — | — | 있음 | 거의 없음 |
| 모델 파라미터 수 | 가장 적음 | 중간 | 중간 | 가장 많음 |
| 추론 시 Flow 실행 수 | 1번 | 1번 | 1번 | 2번 (z 생성 + 행동 생성) |

## 9. 평가 지표 — 어떻게 성능을 측정하나요?

100 에폭 학습 후, 검증 데이터(전체의 10%)에서 다음 지표를 측정합니다.

---

### 지표 1: Action MSE (Prior)

**뭔가요?** 모델이 "혼자서" (정답 행동 없이 Prior만 사용해서) 예측한 행동과 정답 행동 사이의 평균 제곱 오차(Mean Squared Error)

**낮을수록 좋습니다.**

```
action_mse_prior = 평균( ||예측_행동 - 정답_행동||² )
```

---

### 지표 2: Action MSE (Posterior)

**뭔가요?** "정답 행동을 미리 알려줬을 때" (Posterior 사용) 예측한 행동과 정답의 MSE

**낮을수록 좋고, 이론상 Prior MSE의 하한값입니다 (upper bound of quality).**

```
prior_posterior_gap = action_mse_prior - action_mse_posterior
```

이 차이가 작을수록 Prior가 Posterior를 잘 흉내 낸다는 뜻입니다.

---

### 지표 3: Best-of-K (K=1, 3, 5, 10)

**뭔가요?** 확률론적 모델(M3, M4)에서 K번 샘플링해서, 그 중 정답과 가장 가까운 것을 고를 때의 MSE

**낮을수록 좋고, K가 클수록 좋아질 가능성이 높습니다.**

```
best_of_5 = 평균( min(MSE(샘플1, GT), MSE(샘플2, GT), ..., MSE(샘플5, GT)) )
```

**왜 중요한가요?** 실제 로봇 시스템에서는 몇 가지 행동 후보를 생성하고, 그 중 가장 안전하거나 적절한 것을 선택하는 방식을 쓸 수 있습니다. 이 지표가 그 상황을 시뮬레이션합니다.

best_of_1 대비 best_of_5가 얼마나 개선되냐 = 다양성의 실제 효용

---

### 지표 4: Future Cosine Similarity

**뭔가요?** z로부터 예측한 "미래 이미지 임베딩"이 실제 미래 이미지 임베딩과 얼마나 비슷한가

**1에 가까울수록 좋습니다.**

```
future_cosine_sim = cosine_similarity(predict(z), actual_future_embed)
```

이 값이 높다면, z가 "앞으로 어떤 상태가 될지"를 잘 인코딩하고 있다는 뜻입니다.

---

### 지표 5: Sampling Diversity

**뭔가요?** 같은 상황에서 여러 번 샘플링했을 때, 나온 행동들이 얼마나 다른가

**너무 낮으면 다양성이 없고, 너무 높으면 제어 불가능할 수 있습니다.**

```
sampling_diversity = std(행동_샘플_1, 행동_샘플_2, ..., 행동_샘플_N)
```

---

### 지표 6: z-Shuffle Gap ⭐ (핵심 지표)

**뭔가요?** 배치 안에서 z를 섞어버렸을 때 성능이 얼마나 떨어지나

```
z_shuffle_gap = MSE(z를_섞은_경우) - MSE(정상_경우)
```

**클수록 z가 더 중요하다는 뜻입니다.**

**왜 이게 중요한가요?**

z가 실제로 의미 있는 정보를 담고 있다면, A라는 상황용 z를 B라는 상황에 넣었을 때 행동이 엉망이 되어야 합니다. 반대로 z가 아무 의미 없는 노이즈라면 (Posterior Collapse 상태), 섞어도 성능이 별로 안 떨어집니다.

이 지표로 "Posterior Collapse가 일어났는지"를 간접적으로 확인할 수 있습니다.

### 주요 결과 해석

#### "z-Shuffle Gap = 0.72"가 의미하는 것

z를 섞었을 때 MSE가 0.72만큼 올라간다는 것은, **정상적인 MSE의 약 190%** 만큼 성능이 나빠진다는 뜻입니다. 이는 z가 매우 중요한 정보를 담고 있다는 강력한 증거입니다.

만약 Posterior Collapse가 일어났다면, z를 섞어도 성능이 거의 안 떨어질 것입니다. 이 높은 값은 "StochFlowPrior에서 Posterior Collapse가 일어나지 않았음"을 보여줍니다.

#### "Best-of-5 = 0.2421 vs Best-of-1 = 0.3863"이 의미하는 것

5번 샘플링해서 최선을 고르면 MSE가 **0.3863 → 0.2421**로, 약 37% 줄어듭니다. 이건 모델이 진짜로 다양하고 유효한 행동 후보들을 생성하고 있다는 뜻입니다.

비교: FlatFlow나 DetLatent는 매번 거의 같은 행동이 나오므로, Best-of-5 ≈ Best-of-1이어야 합니다.

#### "Future Cosine Sim = 0.9999"가 의미하는 것

z로부터 미래 이미지 임베딩을 거의 완벽하게 예측합니다. z가 "앞으로 어떤 상태에 도달할지"를 담고 있다는 뜻입니다. 즉 z는 단순한 행동 패턴이 아니라 **목표 지향적(goal-conditioned) 정보**를 인코딩합니다.

#### Temperature Trade-off

- **Temperature를 낮추면** (0.3~0.5): 샘플링 분산이 줄어들어 더 "확실한" 행동이 나옴. 하지만 MSE가 항상 줄어드는 건 아님 — 오히려 prior 분포의 Mode에서 벗어난 방향으로 붕괴되기도 함
- **Temperature 1.0이 최선**: 다양성도 좋고 MSE도 괜찮음. Best-of-5에서 가장 낮은 MSE(0.2421)를 달성

**의의:** 계획(z)에 필요한 최소한의 정보가 뭔지 파악. 실제 로봇 배포 시 센서 선택에 활용 가능.

## 12. 전체 인사이트 정리

### 인사이트 1: 잠재 변수는 반드시 필요하다

FlatFlow처럼 z 없이 직접 행동을 생성하면, 다양한 해결 방법 중 "평균"에 해당하는 행동을 배울 위험이 높습니다. 실제 로봇 조작에서는 하나의 상황에 여러 유효한 행동 방식이 있으므로, z를 통해 "어떤 방식으로 할 것인지"를 분리해야 합니다.

### 인사이트 2: Flow Prior가 Gaussian Prior보다 유리하다

VAE(M3)의 Gaussian Prior는 단순하지만 Posterior Collapse 위험이 있습니다. Flow Prior(M4)는 임의의 복잡한 분포를 표현할 수 있고, KL 없이도 Prior-Posterior 정렬을 달성합니다. z_shuffle_gap 값도 더 높아, z가 더 의미 있는 정보를 담을 가능성이 높습니다.

### 인사이트 3: 시맨틱 보조 손실은 초기 학습 안정화에 기여한다

시맨틱 손실은 학습 초반(에폭 1~5)에는 전체 손실의 큰 부분을 차지하지만 이후 거의 0이 됩니다. 이를 통해 초반에 z가 의미 있는 방향으로 학습되도록 가이드합니다. "미래 상태 예측"이 일종의 초기화 역할을 합니다.

### 인사이트 4: Temperature 1.0이 다양성과 정확도의 최적 균형점

낮은 Temperature가 항상 좋은 것은 아닙니다. 모델의 Prior 분포는 Temperature 1.0으로 학습되었으므로, Temperature를 낮추면 Prior 분포의 저밀도 영역을 샘플링하게 되어 오히려 성능이 나빠질 수 있습니다.

### 인사이트 5: 2단계 샘플링은 추론 비용을 두 배로 늘리지만 가치가 있다

M4는 z를 생성하는 Flow(10스텝)와 행동을 생성하는 Flow(10스텝), 총 20스텝이 필요합니다. M1~M3는 10스텝이면 됩니다. 그러나 Best-of-5를 활용하면 MSE가 37% 줄어들므로, 이 추가 비용은 충분히 합리적입니다.

#### z_dim Ablation 전 범위 완성

현재 32, 64, 128이 있고 256이 없습니다. z_dim을 계속 키우면 성능이 계속 좋아지는지, 아니면 어느 지점에서 포화(saturation)되는지 확인해야 합니다.

**왜 중요한가?** 최소한의 z_dim을 파악하면 모델 크기를 줄이면서 성능을 유지할 수 있습니다.

---

#### Best-of-K 곡선 확장

현재 K=1,3,5,10을 측정했습니다. K=20, 50까지 늘려보면 "언제부터 추가 샘플이 도움이 안 되는지(다양성 포화 지점)"를 알 수 있습니다.

---

### 중기 실험

#### 온라인 평가 — 실제 태스크 성공률

현재 평가는 모두 **오프라인 평가(Offline Evaluation)** 입니다. 즉, 데이터셋의 행동과 MSE를 비교하는 것이에요.

하지만 진짜 중요한 것은: **로봇이 실제로 작업을 성공하는가?** 입니다.

오프라인 MSE가 낮아도 실제 성공률이 낮을 수 있고, 반대로 MSE가 약간 높아도 로봇이 잘 작동할 수 있습니다. Robosuite나 LIBERO 시뮬레이터에서 실제 작업 성공률(Task Success Rate)을 측정하는 것이 최종 목표입니다.

---

#### Diffusion Prior vs Flow Prior 비교

M4의 Prior로 DDPM(Denoising Diffusion Probabilistic Model)을 사용하면 어떻게 될까요? Flow Matching Prior와 Diffusion Prior의 성능, 속도, 안정성을 비교합니다.

---

#### SigLIP Fine-tuning

현재는 SigLIP을 frozen(고정)해서 쓰지만, 학습을 허용하면 어떻게 될까요? 데이터에 맞게 미세조정된 시각-언어 특징이 행동 예측을 개선할 수 있을까요?

단, 이는 계산 비용이 크게 늘어나므로 부분적 fine-tuning(adapter 등)도 고려합니다.

---

### 장기 실험

#### 실로봇 배포

시뮬레이터에서 좋은 성능을 보이는 모델을 실제 6-DoF 로봇 팔에서 테스트합니다. 시뮬레이터와 실제 환경의 차이(Sim-to-Real Gap)를 극복하는 것이 핵심 과제입니다.

---

#### 잠재 공간 해석 (Latent Space Visualization)

z_dim=2나 3으로 줄이거나, t-SNE/UMAP으로 128차원 z를 2D로 투영하면:
- 서로 다른 태스크의 z가 클러스터링 되는가?
- 비슷한 행동 패턴은 z 공간에서 가까운가?
- z 공간에서 보간(interpolation)하면 연속적인 행동 변화가 나타나는가?

이런 분석이 가능합니다. z가 의미 있는 구조를 갖추고 있다면, 로봇 계획의 해석 가능성(interpretability)도 높아집니다.

---

#### Foundation VLA와 결합

OpenVLA, Octo 같은 대형 Vision-Language-Action 모델에 StochFlowPrior 헤드를 붙이는 실험입니다. 대형 모델의 일반적인 이해 능력과 StochFlowPrior의 다양성 생성 능력을 결합하면 어떨까요?

## 마무리

이 연구의 핵심을 한 문장으로 요약하면:

> **"로봇이 다양하고 유연하게 행동하려면, 행동을 직접 만들기 전에 '어떤 방식으로 할지'에 대한 잠재 계획 z를 Flow Matching으로 만드는 것이 효과적이다."**

FlatFlow(M1) → DetLatent(M2) → StochVAE(M3) → StochFlowPrior(M4) 순서로 설계가 점점 복잡해지지만, 그만큼 표현력과 다양성이 늘어납니다. 특히 M4는 KL 손실 없이도 Posterior Collapse를 피하면서 의미 있는 잠재 공간을 학습한다는 점이 주목할 만합니다.

앞으로의 실험들이 이 설계 선택들의 효과를 더 명확히 밝혀줄 것입니다.
